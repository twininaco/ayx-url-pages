# ayx-url-pages

本项目用于归档和发布多个独立的 HTML 页面。这些页面可以是前端演示、静态工具、信息展示页或其它无需后端支持的网页文件。

## 项目简介

`ayx-url-pages` 是一个简洁的页面托管仓库，主要功能是将分散的 HTML 页面集中存放，并通过 GitHub Pages 等静态托管服务进行访问。仓库本身不绑定任何特定域名或网站，仅作为页面文件的存储与分发点。

## 目录结构

```
.
├── index.html          # 首页 / 导航页（可选）
├── pages/              # 存放所有独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有归档的 HTML 页面均放置在此目录下，每个文件为一个独立页面。
- `assets/`：存放页面共享的样式、脚本和图片资源，避免重复引用。
- `index.html`：可选，作为仓库根目录的入口页面，可列出所有归档页面的链接。

## 页面归档说明

- 所有页面均为纯前端实现，不依赖后端服务。
- 页面之间相互独立，无数据共享或依赖关系。
- 归档内容应保持简洁、清晰，不包含恶意脚本、跟踪代码或敏感信息。
- 如需新增页面，请直接在 `pages/` 下创建新的 HTML 文件，并确保引用资源路径正确。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/ayx-url-pages.git
   ```
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 如需启用 GitHub Pages，请在仓库设置中开启，并选择 `main` 分支或 `docs/` 文件夹作为源。
4. 访问 `https://your-username.github.io/ayx-url-pages/` 即可查看首页或具体页面。
